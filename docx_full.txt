[P23] Style='Heading 1'
     TEXT: 1. Introduction

[P24] Style='Normal'
     TEXT: Smart contracts are self-executing programs deployed on blockchain networks like Ethereum. Once deployed, they cannot be modified, making security vulnerabilities particularly dangerous. High-profile exploits such as the DAO hack ($60 million loss) and the SpankChain attack ($1 million loss) demonstrate the critical need for automated vulnerability detection tools.

[P25] Style='Normal'
     TEXT: SolidGuard is an ML-powered security analysis tool that detects vulnerabilities in Solidity smart contracts before deployment. The system uses a Random Forest classifier trained on 33 engineered features to identify four major vulnerability types: Reentrancy, Integer Overflow/Underflow, Bad Randomness, and Dangerous Delegatecall.

[P26] Style='Heading 2'
     TEXT: 1.1 Objectives

[P27] Style='List Bullet'
     TEXT: Develop an automated vulnerability detection system for Solidity smart contracts

[P28] Style='List Bullet'
     TEXT: Achieve production-ready accuracy (>90%) using machine learning techniques

[P29] Style='List Bullet'
     TEXT: Create a user-friendly web interface for real-time contract analysis

[P30] Style='List Bullet'
     TEXT: Provide actionable recommendations for fixing detected vulnerabilities

[P31] Style='Heading 2'
     TEXT: 1.2 Scope

[P32] Style='Normal'
     TEXT: This project focuses on static analysis of Solidity source code using pattern-based feature extraction and supervised machine learning. The system currently detects 4 vulnerability types with ~97.00% accuracy and provides heuristic-based detection for 10 additional types.

5. Results returned as JSON with severity, recommendation, and feature breakdown

[P150] Style='Heading 1'
     TEXT: 8. Dataset Expansion Strategy (SmartBugs Wild)

[P151] Style='Normal'
     TEXT: To achieve the ultimate goal of detecting 9+ vulnerability types, the project will expand its dataset by leveraging the "SmartBugs Wild" repository. This dataset contains 47,398 real-world Ethereum smart contracts.

[P152] Style='Normal'
     TEXT: Because the raw contracts are unlabeled, a "Pseudo-Labeling" (Semi-Supervised) approach will be implemented:
1. Randomly sample 1,000 - 2,000 contracts from the 47k pool.
2. Process them using the mature heuristic `feature_extractor.py` to extract features and assign pseudo-labels.
3. Append this massive volume of new data to `features.csv` to drastically increase the model's vocabulary and resilience across 9+ classes.
